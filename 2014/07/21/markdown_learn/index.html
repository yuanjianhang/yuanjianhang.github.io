Markdown介绍

首先，markdown是做什么的？维基百科上说：markdown允许人们“使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档”。说白了，就是你可以用markdown语法写文章，然后就会有一些工具自动帮你把它转换成xhtml（或html）文档，你的文章就可以发表在使用最广泛的web互联网上了。那为什么要绕这个弯子，不直接写html呢？答案肯定是markdown比html更方便啦(好像是废话了…)，markdown更能专注于创作本身。

关于markdown的一些概念请自己google。这里附上wiki地址。

怎样看这篇笔记

我个人建议打开一个markdown编辑器，一边看一边敲代码，估计半个小时用不了就明白了。编辑器一定要支持实时预览功能，mac平台下有macDown，其它平台诸君自己搜索吧。

Markdown 语法

段落

在markdown中，段落文本之间间隔一个空行就会自动分割成两个段落。

标题

接触过web的同学应该都知道html元素h1,h2……h6代表什么吧。没有接触过web的同学看看这篇文献那些黑色的大文字，那就是标题元素。

标题元素总共有6级，如下：

# 这个代表一级标题
## 这个代表二级标题
### 这个代表三级标题
...

完事，几个#号就代表几级标题的意思，不同标题会有不同的字体大小等样式的区别，这个写个代码一看就明白。如果就想#号当成普通的符号使用，该如何呢？

特殊字符处理

处理特殊字符，就用反斜杠(\)，反斜杠后面的字符就会作为普通字符显示，下面的符号都可以和反斜杠搭配：

\    反斜杠
`   反引号
*   星号
_   底线
{}  花括号
[]  方括号
()  括弧
#   井字号
+   加号
-   减号
.   英文句点
!   惊叹号
